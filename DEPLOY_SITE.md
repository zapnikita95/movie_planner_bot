# Деплой сайта movie-planner.ru

## Кто деплоит
**Коммиты и пуш по сайту и боту делает агент (Cursor).** Не нужно пушить вручную после правок — агент сам сделает `git add`, `git commit`, `git push`.

## Где лежит код сайта
- В этом репо: **`moviebot/site/`** (и дубликат **`movie_planner_site/`** в корне).
- Актуальная версия для лендинга — в `moviebot/site/` (index.html, app.js, style.css, CNAME, images/).

## Как сейчас хостится сайт
- Если сайт на **отдельном репо** (например `zapnikita95/movie_planner_site`) и оттуда идёт GitHub Pages → этот пуш в `movie_planner_bot` на сайт не влияет, всё ок.
- Если сайт хостится из **этого репо** (movie_planner_bot) через GitHub Pages:
  - Pages умеет отдавать только **корень** репо или папку **`/docs`**.
  - Сайт в `moviebot/site/` сам по себе не отдаётся.
  - Нужно либо:
    1. В настройках репо: **Settings → Pages → Source**: Branch `main`, папка **`/docs`**, и положить копию сайта в папку `docs/` в репо; либо
    2. Продолжать использовать отдельный репо только для сайта и деплоить сайт оттуда.

## После правок
После любых правок в `moviebot/site/` или в бэкенде (web_app и т.д.) деплой делает агент: коммит + пуш в нужный репо (этот или отдельный сайтовый — как настроено у тебя).
